# Contributing

## Project Purpose

This repository was created as a personal infrastructure and cybersecurity lab focused on learning and portfolio development.

---

## Guidelines

Before publishing any configuration or documentation:

- Remove sensitive information
- Sanitize IP addresses
- Remove credentials and certificates
- Avoid exposing personal environment details

---

## Documentation Standards

All troubleshooting and implementation notes should include:

- Problem description
- Root cause
- Solution applied
- Validation steps

---

## Repository Structure

Keep documentation organized inside their respective folders:

- `/firewall`
- `/vpn`
- `/monitoring`
- `/architecture`
- `/troubleshooting`

---

## Goals

The main purpose of this repository is to:
- Demonstrate infrastructure knowledge
- Practice troubleshooting
- Document real-world scenarios
- Improve technical documentation skills
